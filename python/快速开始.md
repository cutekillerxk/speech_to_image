# 快速开始指南

## 🚀 一键启动（最简单）

### Windows用户

1. **双击运行** `启动.bat` 文件
2. 等待浏览器自动打开
3. 开始使用！

> 脚本会自动检查环境、安装依赖、启动应用

---

## 📦 手动启动（如果一键启动失败）

### 步骤1：创建Conda环境

```bash
conda env create -f environment.yml
```

或者手动创建：

```bash
conda create -n ati python=3.10
conda activate ati
```

### 步骤2：安装依赖

```bash
cd python
pip install -r requirements.txt
```

### 步骤3：配置API密钥（可选）

1. 复制 `.env.example` 为 `.env`
   ```bash
   copy .env.example .env
   ```

2. 编辑 `.env` 文件，填入你的豆包API密钥：
   ```env
   DOUBAO_API_KEY=your_api_key_here
   ```

> **注意**：如果没有API密钥，应用会使用模拟模式（显示占位图片），可以用于测试界面功能。

### 步骤4：启动应用

```bash
python app.py
```

应用会自动在浏览器中打开：http://localhost:7860

---

## ✅ 验证安装

启动后，你应该看到：

1. ✅ 控制台显示 "🚀 启动应用..."
2. ✅ 浏览器自动打开Gradio界面
3. ✅ 界面显示 "👋 欢迎使用！输入文字描述，点击生成按钮开始吧～"

如果看到这些，说明安装成功！

## ❓ 常见问题

### Q: 提示"conda不是内部或外部命令"
A: 需要先安装Anaconda或Miniconda，并添加到系统PATH

### Q: 提示"找不到模块"
A: 运行 `pip install -r requirements.txt` 安装依赖

### Q: 端口被占用
A: 修改 `app.py` 中的 `server_port=7860` 为其他端口（如7861）

### Q: 浏览器没有自动打开
A: 手动访问 http://localhost:7860

### Q: API调用失败
A: 
- 检查 `.env` 文件中的API密钥是否正确
- 如果没有API密钥，这是正常的，系统会使用模拟模式
